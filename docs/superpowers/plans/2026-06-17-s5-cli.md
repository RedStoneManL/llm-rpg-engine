**Goal:** Make the engine playable from a terminal: wire all systems + store + provider + embedder, a minimal new-game genesis, and a REPL play loop (input â†’ turn â†’ player-facing narration), with OOC commands. Offline-tested with `FakeLLMProvider`; live play uses real `make_provider` (env keys).

**Architecture:** `app/` package. `build_engine(campaign_dir, *, provider=None, embedder=None)` constructs the registry (all 6 systems, ontology first), the on-disk event store (`open_store` with `registry.event_types()`), the provider (passed-in or `make_provider` from env, default `FakeLLMProvider` if none), and the embedder (`engine.embed.get_embedder()`). `new_game(engine, *, provider)` seeds a protagonist + starting place. `play_loop` reads input lines, dispatches OOC commands (lines starting `/`) or runs a turn (`run_turn` with `AuthorStrategy`, or `run_compare` when toggled) and prints `result.narration`. The loop is a pure generator/function over an input iterable so it's testable offline by feeding scripted inputs + a `FakeLLMProvider`.

**Tech Stack:** Python 3.12 stdlib; reuses everything (kernel/systems/llm/memory/context/loop). Offline tests `python3 -m pytest -q --ignore=tests/test_embed_real.py`.

**Conventions/Guardrails:** TDD per task; `get_logger("app.*")`; commit per task; `python3`. Never `git init`/`rm -rf .git`/`checkout --orphan`; never delete/modify `_legacy/` or `docs/`; new package `app/`; import others read-only.
